# 测试策略与流程（模块测试 + 统一测试）

## 1. 目标

1. 每个功能模块都能在不依赖其他模块的情况下被单独测试（单测/模块测试）。
2. 支持统一测试入口，便于验收与 CI。
3. 预留端到端测试，覆盖关键业务链路（申请→审核→报名→签到→结项）。

## 2. 测试分层

### 2.1 单元/模块测试（Vitest）

- 后端：路由与服务层可用 `createApp()` + mock repo 的方式测试。
- 前端：组件/页面用 React Testing Library 测试渲染与交互。
- shared：schema/工具函数测试。

统一命令：

```bash
pnpm -r test:run
```

### 2.2 端到端测试（Playwright，预留）

- 建议后续把关键流程写成 e2e：登录、发起申请、管理员审核、报名、签到、结项。
- 命令（后续完善）：`pnpm --filter @campus-activity/web test:e2e`

## 3. 最小可行测试用例（当前骨架）

- 后端：`GET /api/v1/health` 返回 `200` 与 `ok`。
- 前端：首页渲染成功，显示“骨架已启动”提示。
